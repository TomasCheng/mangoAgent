## 核心类关系

```
class Agent
    ├── worktree_manager: WorktreeManager
    ├── task_manager: TaskManager
    ├── event_bus: EventBus
    ├── tools: Dict[str, Tool]
    └── skills: Dict[str, Skill]

class WorktreeManager
    ├── worktrees: Dict[str, WorktreeInfo]
    ├── events: List[WorktreeEvent]
    └── bind_task(worktree_name, task_id)

class TaskManager
    ├── tasks: Dict[int, Task]
    ├── next_task_id: int
    └── update_status(task_id, status)

class EventBus
    ├── subscribers: Dict[str, List[Callable]]
    ├── publish(event_type, data)
    └── subscribe(event_type, callback)

class Tool (基类)
    ├── name: str
    ├── description: str
    └── execute(**kwargs) -> Any
```

## 工作流程示例

### 1. 创建任务并绑定工作树
```
用户: task_create("开发功能X", "实现用户登录")
系统: 创建任务ID=1
用户: worktree_create("login-feature", task_id=1)
系统: 创建工作树并绑定到任务1
用户: worktree_run("login-feature", "git status")
系统: 在login-feature工作树中执行git status
```

### 2. 团队协作
```
用户: spawn_teammate("frontend", "前端开发", "负责UI实现")
系统: 创建前端开发成员
用户: send_message("frontend", "请实现登录页面")
系统: 发送消息给前端成员
用户: broadcast("明天上午开会讨论进度")
系统: 广播消息给所有成员
```

### 3. 技能应用
```
用户: load_skill("git_workflow")
系统: 加载Git工作流技能
用户: 执行Git相关操作时自动应用最佳实践
```

## 扩展点

### 1. 添加新工具
- 在 `src/mangoAgent/tools/` 下创建新工具类
- 继承 `Tool` 基类
- 实现 `execute` 方法
- 在 `__init__.py` 中注册

### 2. 添加新技能
- 在 `.mango/skills/` 或 `work-space/skills/` 下创建技能目录
- 创建 `SKILL.md` 文件描述技能
- 使用 `load_skill()` 加载技能

### 3. 自定义事件处理器
- 订阅事件总线上的事件
- 实现事件处理逻辑
- 在系统初始化时注册处理器

## 总结

MangoAgent 采用分层架构设计，核心模块职责清晰，数据流明确。系统具有良好的扩展性和可维护性，支持多种工作场景和协作模式。
